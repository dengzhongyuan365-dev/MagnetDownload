## 🔧 CEF 集成需求分析

### CEF 版本选择

| CEF 版本 | Chromium 版本 | 发布时间 | 推荐度 | 说明 |
|----------|---------------|----------|--------|------|
| CEF 118 | Chromium 118 | 2023-10 | ⭐⭐⭐⭐⭐ | **推荐** - 最新稳定版，支持现代 Web 标准 |
| CEF 114 | Chromium 114 | 2023-06 | ⭐⭐⭐⭐ | 稳定版本，广泛使用 |
| CEF 110 | Chromium 110 | 2023-02 | ⭐⭐⭐ | 较老版本，兼容性好 |

- 更好的性能和安全性
- 支持现代 CSS 特性 (Container Queries, :has() 等)

### CEF 平台特定需求

#### Windows 平台
- **依赖**: Visual Studio 2019+ Runtime
- **架构**: x64/x86 支持
- **特殊处理**: 
  - Windows 消息循环集成
  - DPI 感知设置
  - 窗口图标和任务栏集成

#### Linux 平台
- **依赖**: GTK3/X11 库
- **架构**: x64/ARM64 支持
- **特殊处理**:
  - X11 窗口管理
  - 字体渲染配置
  - 系统主题集成

#### macOS 平台
- **依赖**: macOS 10.15+ 系统框架
- **架构**: x64/ARM64 (Apple Silicon) 支持
- **特殊处理**:
  - Cocoa 应用程序包结构
  - 沙盒权限配置
  - 原生菜单栏集成

## 🏗️ 架构设计原则

### 1. 分层架构
```
┌─────────────────────────────────────────┐
│           前端表现层 (React)              │  ← 用户界面、交互逻辑
├─────────────────────────────────────────┤
│           桥接通信层 (CEF Bridge)         │  ← JS ↔ C++ 双向通信
├─────────────────────────────────────────┤
│           业务逻辑层 (C++ Controllers)    │  ← 下载管理、状态控制
├─────────────────────────────────────────┤
│           协议处理层 (BitTorrent)         │  ← DHT、Peer、Tracker
├─────────────────────────────────────────┤
│           系统接口层 (OS APIs)            │  ← 文件系统、网络
└─────────────────────────────────────────┘
```

### 2. 模块化设计
- **前端模块**: 按功能域划分 (下载管理、设置、统计等)
- **通信模块**: 统一的 API 接口定义
- **后端模块**: 保持现有的模块化结构

### 3. 数据流设计
```
用户操作 → React 组件 → Zustand Store → CEF Bridge → C++ Controller
                                                            ↓
C++ 事件 ← CEF Bridge ← 状态更新 ← Zustand Store ← React 组件
```

### 4. 错误处理策略
- **前端**: 全局错误边界 + 局部错误处理
- **通信**: 超时重试 + 错误码映射
- **后端**: 异常安全 + 日志记录
